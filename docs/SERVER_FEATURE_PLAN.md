# PulseVita 服务端功能规划

## 思维导图

```

        │         │          │          │          │

        │         │          │          │          │
```

## 一、数据中枢层

### 1.1 多设备数据同步
- **现状**：数据仅存本地 SQLite，换手机数据丢失
- **方案**：服务端作为数据中枢，多设备通过 JWT 认证后增量同步
- **价值**：用户换机无忧，多设备数据一致

### 1.2 云端备份与恢复
- **方案**：每日自动备份用户数据到服务端，支持一键恢复
- **价值**：数据安全感，防止意外丢失

### 1.3 数据导出
- **方案**：支持导出为 CSV / PDF 格式的健康报告
- **价值**：用户可以分享给医生或家人

### 1.4 数据统计 API
- **方案**：服务端聚合计算周/月/季度统计数据
- **价值**：减轻 APP 端计算压力，支持更复杂的分析

## 二、AI 智能层

### 2.1 每日优化建议（已确认）
- AI 分析用户近 7 天数据，生成 3 条个性化建议
- 用户投票，最高票加入优化计划

### 2.2 周/月健康报告
- **方案**：每周日 / 每月 1 号，AI 自动生成健康分析报告
- **内容**：
  - 本周/月数据总览（饮水达标率、运动频率、心情趋势）
  - 与上周/月对比（进步/退步的维度）
  - AI 综合评价 + 下阶段重点建议
- **价值**：用户看到自己的变化趋势，增强粘性

### 2.3 异常预警
- **方案**：AI 检测到异常模式时主动提醒
  - 连续 3 天心情评分 < 2 → "最近心情不太好，要不要试试呼吸练习？"
  - 连续 5 天未运动 → "好几天没运动了，今天走走路？"
  - 饮水量骤降 50% → "最近喝水少了很多，注意补水"
- **价值**：从被动记录变为主动关怀

### 2.4 智能目标调整
- **方案**：AI 根据用户完成情况动态调整目标
  - 连续 7 天轻松达标 → 建议提高目标
  - 连续多天未达标 → 建议降低目标，避免挫败感
- **价值**：目标始终"跳一跳够得着"

### 2.5 饮食智能分析
- **方案**：用户输入饮食描述（如"中午吃了一碗牛肉面"），AI 估算热量和营养成分
- **价值**：不需要手动查热量，降低记录门槛

## 三、社交互动层

### 3.1 健康挑战（升级）
- **现状**：本地随机挑战，无互动
- **方案**：服务端管理挑战池，支持：
  - 每日挑战（服务端统一推送）
  - 周挑战（持续 7 天，如"连续 7 天喝够 2000ml"）
  - 挑战完成率排行
- **价值**：挑战有参与感，不是"一个人在做"

### 3.2 好友系统
- **方案**：用户可以添加好友，查看好友的：
  - 打卡天数
  - 今日是否完成饮水/运动目标
  - 正在进行的挑战
- **注意**：只展示完成状态，不展示具体数据（保护隐私）
- **价值**：社交监督，互相激励

### 3.3 排行榜
- **方案**：按周统计，排行维度：
  - 连续打卡天数
  - 运动时长
  - 饮水达标天数
- **价值**：轻度竞争，提升活跃度

### 3.4 成就系统（服务端）
- **现状**：本地成就，服务端可管理更丰富的成就
- **方案**：
  - 铜/银/金三级成就
  - 稀有成就（如"连续打卡 100 天"）
  - 成就徽章展示在个人主页
- **价值**：长期目标感，用户留存

## 四、内容运营层

### 4.1 动态健康贴士
- **现状**：本地固定 10 条贴士
- **方案**：服务端管理贴士库，支持：
  - 按季节/节气自动推送
  - 按用户健康数据匹配贴士
  - 运营可后台增删改
- **价值**：内容常看常新

### 4.2 节气养生专题
- **方案**：每个节气自动推送专题内容
  - 节气介绍 + 养生要点 + 推荐食谱 + 推荐运动
  - 配合 AI 个性化调整
- **价值**：中医特色，差异化竞争力

### 4.3 健康知识库
- **方案**：按类别组织的健康知识文章
  - 饮食营养 / 运动科学 / 心理健康 / 中医养生
  - 用户可收藏、点赞
- **价值**：从工具升级为健康平台

## 五、用户体验层

### 5.1 智能提醒（服务端推送）
- **方案**：服务端根据用户习惯，选择最佳时间推送提醒
  - 分析用户活跃时段 → 在活跃前 30 分钟推送
  - 饮水提醒：根据上次喝水时间间隔推送
  - 运动提醒：如果当天还未运动，下午推送
- **暂不做**（用户确认暂不需要推送），但预留接口

### 5.2 个性化主题推荐
- **方案**：根据季节/节日自动推荐主题
  - 春天 → 清新绿色主题
  - 冬天 → 温暖暖杏主题
  - 节日限定主题
- **价值**：新鲜感，视觉体验

### 5.3 用户画像
- **方案**：服务端构建用户健康画像
  - 健康类型：运动型 / 饮食型 / 养生型 / 新手
  - 活跃时段：早起型 / 夜猫型
  - 偏好：喜欢的运动类型、饮食习惯
- **价值**：更精准的个性化服务

### 5.4 连续打卡奖励
- **方案**：
  - 连续 7 天 → 解锁银色徽章
  - 连续 30 天 → 解锁金色徽章 + 专属主题
  - 连续 100 天 → 稀有成就 + 数据报告
- **价值**：游戏化激励

## 六、技术架构升级

### 6.1 服务端管理后台
- **方案**：Web 管理后台（Vue 3 + Element Plus）
  - 用户管理（查看用户数、活跃度）
  - 内容管理（贴士、挑战、知识库 CRUD）
  - 数据看板（DAU、留存率、功能使用率）
  - AI 建议管理（查看生成质量、手动调整）
- **价值**：运营可独立管理，不需要改代码

### 6.2 消息队列
- **方案**：引入 RabbitMQ / Redis Stream
  - AI 建议生成异步处理（不阻塞请求）
  - 批量通知推送
  - 数据同步解耦
- **价值**：系统可扩展性

### 6.3 监控与告警
- **方案**：
  - 服务健康监控（Spring Boot Actuator）
  - API 调用量统计
  - 异常告警（邮件/企业微信）
- **价值**：问题早发现早处理

## 八、推荐实施路线

```
第一阶段（3天）：基础能力
  ├── Spring Boot 项目搭建
  ├── MySQL + Redis 环境
  ├── 用户注册/登录 API
  ├── 健康记录同步 API
  └── Docker 部署

第二阶段（2天）：AI 智能
  ├── AI 每日建议生成
  ├── 投票 + 优化计划
  └── AI 周报告生成

第三阶段（2天）：APP 端改造
  ├── 登录/注册页面
  ├── 数据同步逻辑
  ├── 建议卡片 + 投票 UI
  ├── 优化计划页面（「我的」模块）
  └── 首页集成建议

第四阶段（2天）：内容与体验
  ├── 动态健康贴士
  ├── 成就系统（服务端）
  ├── 数据导出
  └── 管理后台（基础版）

第五阶段（持续迭代）
  ├── 好友系统
  ├── 排行榜
  ├── 异常预警
  └── 更多 AI 能力
```
